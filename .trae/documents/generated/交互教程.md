## 🔧 第一步：理解项目结构

基于当前项目的实际结构：

```
autowrKPI/
├── index.html              # 主页面入口
├── css/dashboard.css       # 样式文件
├── js/
│   ├── dashboard.js        # 主控制器
│   ├── data.worker.js      # 数据处理引擎
│   └── static-report-generator.js  # 通信桥接
└── assets/                 # 静态资源
```

## 📊 第二步：理解数据流

基于实际代码的数据处理流程：

1. **数据导入**: CSV文件 → File API → 解析为JSON
2. **数据处理**: 原始数据 → Worker处理 → 聚合计算
3. **数据筛选**: 时间筛选 + 下钻筛选 → 过滤数据
4. **数据可视化**: 聚合数据 → ECharts配置 → 渲染图表
